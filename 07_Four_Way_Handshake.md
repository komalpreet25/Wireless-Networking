# WPA/WPA2 Four-Way Handshake

## Description

The Four-Way Handshake is a secure authentication process used in WPA2 and WPA3 networks to establish encrypted communication between a wireless client (Station) and an Access Point (AP).

Its purpose is **not to send the Wi-Fi password**, but to verify that both devices already know the same password and to generate encryption keys for secure communication.

---

# Why is the Four-Way Handshake Needed?

When a client connects to a Wi-Fi network, both the client and the Access Point must:

* Verify each other's identity
* Generate encryption keys
* Start secure communication
* Prevent unauthorized access

The Four-Way Handshake performs these tasks.

---

# Devices Involved

There are two devices:

```text id="dev1"
Station (Client)  ↔  Access Point (AP)
```

Example:

* Laptop ↔ Wi-Fi Router
* Phone ↔ Access Point

---

# Before the Handshake

Before the handshake starts:

* The client already knows the Wi-Fi password.
* The Access Point also knows the Wi-Fi password.
* Both devices derive the **Pairwise Master Key (PMK)** from that shared password.

The password itself is **never transmitted** during the handshake.

---

# Important Terms

## PMK (Pairwise Master Key)

The PMK is a master key derived from the Wi-Fi password.

It is used to generate session encryption keys.

---

## PTK (Pairwise Transient Key)

The PTK is a temporary session key generated during the Four-Way Handshake.

It protects communication between the client and the Access Point.

A new PTK is created each time a device connects.

---

## Nonce

A **Nonce** is a random number that is used only once.

Purpose:

* Prevent replay attacks
* Ensure every handshake is unique

Two nonces are used:

* ANonce (generated by the Access Point)
* SNonce (generated by the Station)

---

## MIC (Message Integrity Code)

MIC verifies that handshake messages have not been modified during transmission.

It ensures message integrity.

---

# Four-Way Handshake Process

The handshake consists of **4 EAPOL (Extensible Authentication Protocol over LAN) messages**.

---

# Message 1

## AP → Station

The Access Point sends:

* ANonce

```text id="msg1"
Access Point
      │
   ANonce
      │
Station
```

Purpose:

* Start the handshake
* Provide a random number

---

# Message 2

## Station → AP

The Station:

* Generates SNonce
* Calculates the PTK
* Sends:

  * SNonce
  * MIC

```text id="msg2"
Station
   │
SNonce + MIC
   │
Access Point
```

Purpose:

* Prove knowledge of the PMK
* Begin secure key generation

---

# Message 3

## AP → Station

The Access Point:

* Verifies the MIC
* Calculates the PTK
* Sends:

  * Group Temporal Key (GTK)
  * MIC

```text id="msg3"
Access Point
      │
GTK + MIC
      │
Station
```

Purpose:

* Share the group encryption key
* Continue secure setup

---

# Message 4

## Station → AP

The Station sends a final acknowledgment.

```text id="msg4"
Station
   │
ACK
   │
Access Point
```

Purpose:

* Confirm successful key installation
* Complete the handshake

---

# Complete Handshake Diagram

```text id="flow1"
Access Point                     Station

     | -------- Message 1 --------> |
     |        ANonce                |
     |                              |
     | <------- Message 2 --------- |
     |      SNonce + MIC            |
     |                              |
     | -------- Message 3 --------> |
     |       GTK + MIC              |
     |                              |
     | <------- Message 4 --------- |
     |            ACK               |
```

---

# What Happens After the Handshake?

After Message 4:

* Secure connection established
* Data encryption begins
* Wireless communication continues safely

---

# Why the Password Is Not Sent

A common misconception is that the Wi-Fi password is transmitted.

This is incorrect.

Only cryptographic values such as nonces and integrity checks are exchanged.

The password remains on the client and the Access Point.

---

# Why Nonces Are Important

If the same values were reused:

* Attackers could replay old packets
* Security would weaken

Random nonces ensure each handshake is unique.

---

# Group Temporal Key (GTK)

GTK protects **broadcast** and **multicast** traffic.

Example:

```text id="gtk1"
Broadcast Packet
       ↓
Encrypted using GTK
```

---

# Pairwise Transient Key (PTK)

PTK protects **unicast** traffic between one client and the Access Point.

Example:

```text id="ptk1"
Laptop ↔ Access Point
Encrypted using PTK
```

# Summary

* The Four-Way Handshake is used in WPA2 and WPA3.
* It consists of four EAPOL messages.
* The password is **never transmitted**.
* PMK is derived from the shared password.
* PTK encrypts unicast traffic.
* GTK encrypts broadcast and multicast traffic.
* Nonces make every handshake unique.
* MIC ensures message integrity.
* The handshake establishes secure communication between the client and the Access Point.
