# Packet Injection

## Description

Packet Injection is the process of transmitting specially crafted wireless frames through a Wi-Fi adapter instead of only listening to existing traffic.

Unlike Monitor Mode, which primarily captures packets, Packet Injection allows a compatible wireless adapter to **transmit IEEE 802.11 frames** for legitimate network testing, device validation, protocol research, and troubleshooting in authorized environments.

---

# What is Packet Injection?

Normally, a wireless adapter sends only the packets generated by the operating system and applications.

With packet injection support, the adapter can transmit custom wireless frames for testing purposes.

Example:

```text id="pi1"
Wireless Adapter
        │
Creates Wireless Frame
        │
Transmits Frame
        │
Nearby Access Point / Station
```

# Passive vs Active Operations

## Passive Operation

The adapter only listens.

Example:

```text id="passive1"
Nearby Wi-Fi Traffic
        │
Monitor Mode
        │
Packet Capture
```

Characteristics:

* No transmission
* Observation only

---

## Active Operation

The adapter transmits wireless frames.

Example:

```text id="active1"
Wireless Adapter
        │
Creates Frame
        │
Transmits
```

Characteristics:

* Sends wireless frames
* Used for testing and communication

---

# Monitor Mode vs Packet Injection

| Feature                     | Monitor Mode | Packet Injection     |
| --------------------------- | ------------ | -------------------- |
| Captures Packets            | Yes          | No (Primary Purpose) |
| Sends Packets               | No           | Yes                  |
| Passive                     | Yes          | No                   |
| Used for Analysis           | Yes          | Yes                  |
| Used for Frame Transmission | No           | Yes                  |

# IEEE 802.11 Frames

Packet injection works with IEEE 802.11 wireless frames.

Common frame categories include:

* Management Frames
* Control Frames
* Data Frames

These frame types define how Wi-Fi devices communicate.

# Acknowledgment (ACK)

When a wireless device successfully receives a frame, it normally sends an acknowledgment.

Example:

```text id="ack1"
Station
   │
Receives Frame
   │
ACK
   │
Access Point
```

ACK frames improve reliability by confirming successful delivery.

# Common Misconceptions

### Myth

Packet injection works on every Wi-Fi adapter.

**Reality**

Support depends on the adapter chipset and drivers.

---

### Myth

Packet injection automatically improves Wi-Fi speed.

**Reality**

It is a testing capability, not a performance feature.

---

### Myth

Packet injection replaces Monitor Mode.

**Reality**

Monitor Mode and Packet Injection serve different purposes.

Monitor Mode observes traffic.

Packet Injection transmits frames.
